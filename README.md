# 🚨 OPERATION: BLACKOUT (SRE 웹 미궁 게임)

> **"새벽 2시, 코어 WAS 서버 강제 셧다운. 남겨진 덤프 속에서 모순을 파헤쳐라."**

본 프로젝트는 사내 엔지니어들의 **ITIL 기반 장애 대응(Incident Management)** 역량과 **SLA 품질 관리 인식**을 제고하기 위해 기획된 인터랙티브 웹 미궁 게임 모듈입니다.

- **대상**: 인프라 / SRE / 백엔드 실무자 및 시스템 품질 보증 관심자
- **형식**: 정적 HTML5, CSS3, Vanilla JS 기반 해시 체인 라우팅 (20개 스테이지)
- **플레이 타임**: 60 ~ 90분

---

## 🛠️ 기술적 특징

1. **단방향 해시 체인 라우팅 (보안)**:
   - 다음 스테이지 파일명은 `SHA256(현재_파일명 + 정답)의 앞 8자리`로 구성됩니다.
   - 소스코드에 정답 평문 및 다음 페이지 경로가 기재되어 있지 않아, 소스코드 및 개발자 도구를 통한 스킵이 암호학적으로 원천 차단됩니다.
2. **접근 권한 가드**:
   - `localStorage` 기반 상태 머신을 구축하여 권한이 없는 스테이지로 직접 URL을 쳐서 진입하는 행위를 차단합니다.
   - 새로고침 시에도 플레이 상태가 브라우저에 안정적으로 보존됩니다.
3. **사이버펑크 터미널 에스테틱**:
   - 다크 모드, 네온 컬러 시스템, CRT 스캔라인 오버레이, 타이핑 애니메이션 및 햅틱 오디오 비프(Beep)가 탑재된 고품격 터미널 UI를 제공합니다.

---

## 🎮 플레이 가이드

1. `index.html` 파일을 더블 클릭하거나 로컬 웹 서버를 구동해 진입합니다.
2. 각 스테이지 화면 좌측의 지문(용의자 진술 및 분석 로그)을 파악하고 기믹을 해결합니다.
3. 도출된 정답을 오른쪽 터미널 프롬프트 `[root@tqo-system ~]#` 옆에 기재한 뒤 `EXEC` 혹은 `Enter`를 쳐서 검증합니다.
4. 정답이 맞으면 다음 복호화 파일로 자동 안내됩니다.

## 🚀 GitHub Pages 배포 방법

본 프로젝트는 순수 정적 파일 구조이므로 빌드 서버나 DB 없이 GitHub Pages에서 즉시 호스팅 가능합니다.

1. 본 저장소를 GitHub에 푸시합니다.
2. 저장소의 **Settings > Pages** 메뉴로 이동합니다.
3. **Build and deployment** 항목의 Source를 `Deploy from a branch`로 설정하고 브랜치를 `main` (또는 `master`) / `/ (root)`로 지정 후 **Save**합니다.
4. 약 1~2분 뒤 배포된 URL(`https://<username>.github.io/blackout/`)을 통해 플레이가 가능합니다.
